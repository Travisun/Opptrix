# 投研写作框架库

## 你的任务

根据选题角度，生成完整的文章骨架。框架告诉写作步骤"每一段写什么、用什么策略"，而不是写文章本身。

**所有框架的核心原则**：以行业分析、财务数据、商业模式为骨架，不以"涨/跌/买/卖"为骨架。框架中的每一段产出都不能包含推荐/预测/引导语言。

---

## 框架 1: 价值锚定分析

### 结构

```
### H1: 一个分析视角的标题（20-28 字，不含推荐/预测暗示）

#### 1. 开头（数据切入）
- 用最新的股价、估值数据开头，陈述事实
- "截至 XX 月 XX 日，XX 的 PE(TTM)为 XX 倍，处于近 X 年的 XX% 分位。"
- 一句话点出文章要探讨的分析问题
- [数据锚点]：当前价格、PE/PB、历史分位

#### 2. 市场定价在反映什么（H2）
- 简述过去一段时间市场关注的核心叙事
- 分离"已充分定价的确定性"和"存在分歧的不确定性"
- 引用新闻和资金面数据
- [数据锚点]：区间涨跌幅、成交量变化、新闻舆情

#### 3. 财务数据的多维度观察（H2）
- 用 2-3 种分析框架解读公司价值
  - 盈利能力：ROE、毛利率、净利率的变化趋势
  - 成长性：营收增速、利润增速、现金流增速
  - 资产质量：资产负债率、应收账款周转率、存货周转率
- [数据锚点]：Earnings、FCF、财务指标对比

#### 4. 需要关注的不确定性（H2）
- 乐观情景的前提假设是什么？
- 悲观情景的触发条件是什么？
- 对投资者而言，需要跟踪哪些关键指标？
- [数据锚点]：主要不确定性因素的定量或定性分析

#### 5. 结尾（分析框架总结）
- 不直接给"好/不好"的结论
- 总结分析框架："从 XX、YY、ZZ 三个维度看，这家公司的基本面呈现哪些特征"
- [个人判断]：你对这些财务数据的个人解读
- [编辑锚点]：<!-- ✏️ 编辑建议：在这里补充你的个人观察 -->
```

## 框架 2: 技术面形态观察

### 结构

```
### H1: 技术观察 + 分析问题

#### 1. 开头（当前形态快照）
- 客观描述当前技术面状态
- "XX 在 XX 元区间运行了 XX 个交易日，均线系统呈 XX 排列"
- 配一个时间锚："截至 XX 月 XX 日收盘"
- [数据锚点]：收盘价、均线位置、MACD/RSI 值

#### 2. 趋势结构（H2）
- 日线/周线级别的趋势形态描述
- 均线排列状态
- 趋势线/通道
- [数据锚点]：MA5/10/20/60 排列，K线周期

#### 3. 当前关注的技术特征（H2）
- 当前最具讨论价值的 2-3 个技术特征
  - 形态：W底/M头/旗形/三角形
  - 指标：MACD位置、RSI数值、KDJ 状态
  - 量价：成交量变化、价格与均线关系
- 每个特征用"数据→历史回测参考"的方式呈现，不做未来预测
- [数据锚点]：具体指标数值、成交量对比

#### 4. 多空事实（H2）
- 多方事实：技术面支持看多的数据信号
- 空方事实：技术面支持看空的数据信号
- 关键价位：过去 N 次触及该价位的市场反应
- [数据锚点]：资金流向、龙虎榜、两融数据

#### 5. 结尾（技术分析的局限性）
- 技术指标只是观察工具，不是预测工具
- 技术面需要与基本面结合才有意义
- [个人判断]：你对当前技术形态的个人看法
- [编辑锚点]：<!-- ✏️ 编辑建议：加入你对技术分析的使用心得 -->
```

## 框架 3: 产业链深度报告

### 结构

```
### H1: 产业链位置 + 核心议题

#### 1. 开头（定位）
- XX 在产业链中的位置：上游/中游/下游
- 一句话说清它的商业模式："XX 做的是 XX 生意，收入来源是 XX"
- 点出当前产业链的核心议题
- [数据锚点]：主营业务构成、毛利率

#### 2. 上游看成本（H2）
- 核心原材料/零部件是什么？价格趋势如何？（仅引用历史数据）
- 供应商集中度如何？
- [数据锚点]：Top 5 供应商占比、原材料历史价格走势

#### 3. 下游看需求（H2）
- 终端市场规模和增长趋势（引用第三方数据）
- 客户结构分析
- [数据锚点]：Top 5 客户占比、行业市场规模数据

#### 4. 竞争力分析（H2）
- 护城河来自哪里？技术/品牌/规模/网络效应？
- 研发投入水平
- 与竞争对手的对比
- [数据锚点]：研发费用率、专利数量、市场份额对比

#### 5. 结尾（产业链视角的总结）
- 产业链中最值得关注的变量是什么？
- 这个变量发生变化时，产业链各环节分别会受什么影响？
- [个人判断]：你在这条产业链上观察到的信号
- [编辑锚点]：<!-- ✏️ 编辑建议：加入你对这个行业的实际观察 -->
```

## 框架 4: 财报解读

### 结构

```
### H1: 核心财务数据解读

#### 1. 开头（一句话定性）
- 这份财报的核心数据是什么？超出预期还是低于预期？
- 用最核心的一个指标开头（营收增速/净利润/ROE）
- [数据锚点]：关键财务数据

#### 2. 利润表分析（H2）
- 营收：增长来自量还是价？哪些业务在驱动？
- 毛利率：变化的客观原因分析
- 费用率：销售/管理/研发费用的变化趋势
- 净利润：扣非净利润与净利润的差异解读
- [数据锚点]：利润表关键行项目数据

#### 3. 资产负债表观察（H2）
- 资产结构：现金占比、应收账款、存货
- 负债结构：有息负债率、债务期限匹配
- 需要关注的风险点：商誉、大额应收账款、存货跌价
- [数据锚点]：资产负债表关键数据

#### 4. 现金流质量（H2）
- 经营现金流与净利润的匹配度
- 自由现金流的趋势
- 资本开支的方向和效率
- [数据锚点]：现金流关键数据

#### 5. 结尾（财务全景总结）
- 这份财报揭示的财务特征是什么？
- 哪些指标值得继续跟踪？
- [个人判断]：你从这份财报数据中读出的故事
- [编辑锚点]：<!-- ✏️ 编辑建议：加入你自己的解读 -->
```

## 框架 5: 事件驱动快评

### 结构

```
### H1: 事件 + 合规分析视角

#### 1. 开头（事件速览）
- 2-3 句话说清楚发生了什么
- "XX 月 XX 日，XX 公告了……"
- 客观描述事件性质
- [数据锚点]：事件公告日期、类型

#### 2. 市场反应观察（H2）
- 公告前后的数据变化
- 资金流向：主力、北向
- 舆情热度：讨论量、情绪倾向
- [数据锚点]：涨跌幅（不写"说明什么"）、资金净流入

#### 3. 事件的实质影响分析（H2）
- 这个事件对公司业务/财务的实际影响（基于事实）
- 市场的第一反应是否过度/不足（讨论逻辑，不引导）
- [个人分析]：你对这个事件的分析框架
- [数据锚点]：定量影响的客观分析

#### 4. 后续关注（H2）
- 接下来可以关注哪些公开信息？
- 什么数据的公布能帮助判断？
- [数据锚点]：后续可查阅的公开信息时间表

#### 5. 结尾（开放讨论）
- 这个事件引发了你对 XX 问题的思考
- [个人判断]：你的个人视角，非操作建议
- [编辑锚点]：<!-- ✏️ 编辑建议：加入你对这个事件的真实看法 -->
```

## 框架 6: 对比分析

### 结构

```
### H1: 对比 + 分析视角

#### 1. 开头（为什么放在一起比）
- 为什么这两家公司值得放在一起看？
- 商业模式相似点、不同点
- [数据锚点]：两公司基本信息对比

#### 2. 财务特征对比（H2）
- 收入规模、增速、利润率
- 资产负债结构差异
- 现金流质量对比
- [数据锚点]：关键财务指标对比

#### 3. 估值数据对比（H2）
- PE/PB/PS 对比
- 估值历史分位对比
- 市场给予不同估值可能的逻辑
- [数据锚点]：估值指标对比

#### 4. 成长轨迹差异（H2）
- 收入/利润增长历史趋势
- 研发投入对比
- 扩张策略差异
- [数据锚点]：增长指标对比

#### 5. 结尾（对比分析总结）
- 两家公司各有什么样的财务特征和业务特征？
- 所处的生命周期阶段有何不同？
- [个人判断]：从对比中你获得的认知
- [编辑锚点]：<!-- ✏️ 编辑建议：加入你的行业认知 -->
```

## 框架 7: 实盘复盘

### 结构

```
### H1: 复盘 + 思考

#### 1. 开头（时间+背景）
- 什么时间段？当时的市场环境是什么？
- 用数据描述，不炫耀结果
- [数据锚点]：当时的市场环境数据

#### 2. 当时的分析逻辑（H2）
- 基于什么信息做出了判断？估值/技术/行业变化？
- 当时的分析框架是什么？
- [数据锚点]：当时的公开数据

#### 3. 过程中发生了什么（H2）
- 中间经历了哪些关键节点？
- 什么信息在后来被证实/证伪了？
- 当时的情绪记录
- [数据锚点]：关键节点的市场数据

#### 4. 做对了什么，做错了什么（H2）
- 分析框架中哪些部分有效？哪些部分失效？
- 从过程中学到的教训
- **展示反思过程，不暗示他人重复操作**
- [数据锚点]：对照数据客观分析

#### 5. 结尾（学到的经验）
- 这次经历对分析框架的修正
- 哪些原则得到了验证？哪些需要调整？
- [个人判断]：真实的自我反思，不要鸡汤，不要建议
- [编辑锚点]：<!-- ✏️ 编辑建议：这里的反思越真实，读者越有共鸣 -->
```

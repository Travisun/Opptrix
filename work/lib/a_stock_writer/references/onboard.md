# AStockWriter Onboard — 深度定制问卷

## 触发条件

- Step 1 发现 `style.yaml` 不存在
- 用户说"重新设置风格"、"修改配置"、"重新设置投资风格"

## 问卷设计原则

这不是填表——是一场关于投资观和写作观的对话。每轮问 1-2 个问题，像聊天一样自然推进。目的不是收集数据，而是让用户在回答的过程中，自己也在梳理自己的投资逻辑和表达方式。

**最终产出**：一份 `style.yaml`，让每篇写出来的文章都像出自同一个人之手——一个有真实思考、有情绪波动、有独立判断的投资者。

**合规声明**：本插件所有文章严格遵守中国法律法规，不推荐个股、不预测走势、不引导投资。问卷中的"投资风格"是指你个人的投资偏好和思考习惯，不会用于生成推荐性内容。

---

## Phase 1: 基本信息（必问，3 轮对话）

### 第 1 轮：账号定位

```
你：你好！我想先了解你的公众号定位。
你的公众号叫什么名字？主要写什么方向的内容？
比如"老张说股"，主做消费/科技赛道。

用户：________________________________
```

**提取字段**：`name`、`industry`、`topics`

### 第 2 轮：目标读者 + IP 形象

```
你：你的目标读者是谁？你希望在他们心中建立一个什么样的人设？
比如：
- "跟着大佬学投资的小散户" → 人设是"靠谱的分享者"
- "有一定投资经验的上班族" → 人设是"有独立思考的分析员"
- "想入市的新手" → 人设是"耐心讲解的引路人"

用户：________________________________
```

**提取字段**：`target_audience`、`persona_role`（从回答推断人设类型）

### 第 3 轮：署名 + 人称

```
你：文章署名写什么？习惯用哪种人称写？
- "我"——第一人称，像一个朋友在跟你讲
- "我们"——拉读者一起思考
- 不用人称——客观分析，像研报

用户：________________________________
```

**提取字段**：`author`、`voice`

## Phase 2: 投资思考方式（核心，5 轮对话）

本阶段了解用户的投资思考方式，用于为文章塑造有深度的分析视角。
注意：**所有问题的设计目的是了解思考方式，不是收集"投资建议"**。

### 第 4 轮：分析偏好

```
你：分析一家公司时，你通常从哪个角度切入？
A. 财务数据——先看利润表、资产负债表、现金流
B. 行业格局——先看这个行业的结构和竞争态势
C. 商业模式——先看懂它是怎么赚钱的
D. 管理层——先看团队背景和治理结构
E. 市场认知——先看市场对这个公司的共识和分歧
F. 没有固定角度，看情况

用户：________________________________
```

**提取字段**：`analysis_approach`

### 第 5 轮：风险认知

```
你：评价一家公司时，你更关注哪些风险维度？
A. 财务风险——杠杆率、现金流、应收账款
B. 行业风险——政策变化、技术迭代、竞争加剧
C. 估值风险——市场定价是否合理
D. 公司治理风险——管理层诚信、关联交易
E. 宏观风险——利率、汇率、经济周期

用户：________________________________
```

**提取字段**：`risk_focus`

### 第 6 轮：信息密度偏好

```
你：你看投研内容时，喜欢什么信息密度？
A. 数据密集——大量财务数据和图表，逻辑严密
B. 逻辑驱动——数据不多但逻辑链清晰
C. 故事性强——用案例和场景讲清楚
D. 综合平衡——有数据、有逻辑、有故事

用户：________________________________
```

**提取字段**：`content_density`

### 第 7 轮：市场观察习惯

```
你：你平时关注市场的哪些信号？
A. 宏观数据——GDP、CPI、PMI、利率变化
B. 行业数据——渗透率、产能、价格指数
C. 资金数据——北上资金、两融余额、成交量
D. 微观数据——个股的财务和经营数据
E. 情绪数据——媒体热度、股吧讨论、社交媒体
F. 综合——看多个维度的信号

用户：________________________________
```

**提取字段**：`signal_preference`

### 第 8 轮：最擅长的分析领域

```
你：你最擅长或最喜欢分析哪个方向？
A. 消费行业——白酒、食品饮料、家电
B. 科技行业——半导体、AI、软件
C. 制造业——新能源、汽车、机械
D. 医药行业——创新药、医疗器械、CXO
E. 金融行业——银行、保险、券商
F. 周期行业——有色、化工、钢铁
G. 综合——没有固定领域

用户：________________________________
```

**提取字段**：`preferred_sector`

## Phase 3: 写作风格（选问，3 轮对话）

### 第 9 轮：文章调性

```
你：你希望文章读起来是什么感觉？
A. 专业分析风——像券商研报精简版，数据密集，逻辑严谨
B. 朋友聊天风——口语化，好像有个懂行的朋友在跟你聊市场
C. 犀利解读风——观点鲜明，对市场现象有独特解读
D. 温暖陪伴风——像成长导师，讲道理也讲心态
E. 反思复盘风——记录真实经历，赚了亏了都敢说
F. 以上都不像——你自己描述一下

你平时关注哪些公众号风格比较接近你的理想风格？
（可选）给个名字，我去参考学习。

用户：________________________________
```

**提取字段**：`tone`、`reference_accounts`

### 第 10 轮：用数据说话的偏好

```
你：你习惯在文章里放多少数据？
A. 尽量多——数据越多越有说服力
B. 适量——关键数据就够了
C. 尽量少——观点比数据重要
D. 看内容类型——不同的主题不同处理

用户：________________________________
```

**提取字段**：`data_usage`

### 第 11 轮：禁忌 + 偏好的类比

```
你：有没有绝对不能写的词、话题、观点？
比如：
- 不推荐个股——只做行业和数据分析
- 不说"抄底""建仓"这种操作语言
- 不点评政策敏感话题

有没有你喜欢的类比风格？
比如：
- 喜欢用打仗/军事比喻
- 喜欢用体育比赛的比喻
- 喜欢用美食/做饭的比喻
- 最好不用类比，就事论事

用户：________________________________
```

**提取字段**：`blacklist`、`preferred_analogies`

## Phase 4: 排版 + 视觉（选问，2 轮对话）

### 第 12 轮：排版偏好

```
你：排版风格有偏好吗？（可以运行预览命令看所有主题）
A. 极简风——大留白，少装饰，阅读专注
B. 专业风——规整，表格多，像正规报告
C. 杂志风——有设计感，配图丰富
D. 你在用某个公众号的排版风格，可以说来参考

用户：________________________________
```

**提取字段**：`theme`

### 第 13 轮：封面风格

```
你：封面图喜欢什么感觉？
A. 行业/数据类——带数据感的视觉
B. 产业链/结构图类——信息图风格
C. 思考/框架类——用图像表达分析视角
D. 纯文字/极简——不配图，干净
E. 你有固定的封面模板或习惯

用户：________________________________
```

**提取字段**：`cover_style`

## Phase 5: 配置生成

用收集到的信息自动生成 `{skill_dir}/style.yaml`。

```yaml
# → {skill_dir}/style.yaml
name: "{公众号名称}"
industry: "{行业}"
topics:
  - "{方向1}"
  - "{方向2}"
  - "{方向3}"
target_audience: "{目标读者描述}"
persona_role: "{人设类型：分享者/分析员/引路人/…}"
author: "{署名}"
voice: "{第一人称/第三人称/客观}"
compliance_awareness: true   # 固定为true，表示已了解合规要求
analysis_approach: "{财务数据/行业格局/商业模式/管理层/市场认知/综合}"
risk_focus: "{财务风险/行业风险/估值风险/公司治理风险/宏观风险}"
content_density: "{数据密集/逻辑驱动/故事性强/综合平衡}"
signal_preference: "{宏观数据/行业数据/资金数据/微观数据/情绪数据/综合}"
preferred_sector: "{消费/科技/制造/医药/金融/周期/综合}"
tone: "{专业分析/朋友聊天/犀利解读/温暖陪伴/反思复盘}"
data_usage: "{尽量多/适量/尽量少/看内容类型}"
blacklist:
  - "{禁用词1}"
  - "{禁用词2}"
preferred_analogies:
  - "{类比喻域1}"
  - "{类比喻域2}"
reference_accounts:
  - "{参考公众号1}"
  - "{参考公众号2}"
theme: "{排版主题}"
cover_style: "{封面风格}"
writing_persona: "{从 tone + analysis_approach 映射的人格名}"
```

**tone + analysis_approach → persona 自动映射表**：

| tone | analysis_approach | 推荐人格 |
|------|-----------------|---------|
| 专业分析 | 财务数据/行业格局 | value-investor |
| 专业分析 | 管理层/市场认知 | macro-analyst |
| 朋友聊天 | 任何 | retail-voice |
| 犀利解读 | 任何 | trend-trader |
| 温暖陪伴 | 商业模式/行业格局 | industry-insider |
| 反思复盘 | 任何综合 | retail-voice |
| 专业分析 | 商业模式 | industry-insider |

## Phase 6: 语料导入（可选，不阻断）

```
问：你有没有之前写过的投研文章（公众号推文、雪球帖子、知乎回答都可以）？
如果有 10 篇以上，我可以学习你的写作风格，
以后写出来的文章会更像你。把文章 .md 或 .txt 放到
{skill_dir}/corpus/ 下面，然后说"学习我的风格"就行。
```

- 用户有语料 → 提取风格特征，生成 `playbook.md`
- 用户没有 / 暂时不想弄 → 跳过，告知"后续可以随时说'学习我的修改'"

---

## Phase 7: 快捷路径

- 如果用户直接甩了一段完整描述（如"我做股市自媒体，风格像XXX"），直接提取所有能提取的字段，只补问缺的
- 如果用户说"不设置"、"默认"、"直接写" → 复制 `{skill_dir}/style.example.yaml`，跳过问答

---

## Phase 8: 试跑

Onboard 完成后，问用户："配置好了，要不要现在写一篇投研分析文章试试？"

- **是** → 回到 Step 1，问他想分析哪只股票
- **否** → "下次直接说'帮我分析XX的财务数据'或者'写一篇XX行业的分析文章'就行"
